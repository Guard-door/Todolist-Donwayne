# 拖拽排序 — 实现计划

**Goal:** 为待办列表实现 Edge 风格拖拽排序（中线判定 + 单元素交换），支持桌面端 DnD 和移动端 Touch。

**Architecture:** 新建独立模块 `modules/todo/features/sortable.js`，通过 `enableSortable(listEl, options)` 接口暴露。主模块 `app.js` 在 `render()` 后调用初始化，通过 `onSortEnd` 回调更新数据。

**Tech Stack:** 原生 HTML5 Drag and Drop API + Touch Events + CSS FLIP 动画

---

## Task 1: 建立文件骨架

- [ ] 创建 `modules/todo/features/sortable.js`（空壳接口 `enableSortable` + `bindSortableItem`）
- [ ] `index.html` 添加 `<script src="modules/todo/features/sortable.js"></script>`
- [ ] 验证页面加载不报错

## Task 2: 桌面端 DnD — 中线判定 + 交换

**状态变量：** `dragId, dragEl, floating, active, lastSwapped`

**核心函数：**
- `crossedMidline(clientY, el)` → `clientY < rect.top + rect.height / 2`
- `createFloating(el, x, y)` → `position:fixed` 克隆，`scale(1.02)` + `box-shadow`
- `performSwap(targetId)` → FLIP：记录 rect → 交换 DOM → transform 补偿 → transition 滑入

**事件（绑定到 listEl，通过 `e.target.closest('.todo-item')` 获取目标）：**
- `dragover` → 中线判定 → `performSwap`
- `drop` → `onSortEnd(newOrder)`
- `dragend`（document）→ cleanup

## Task 3: 移动端 Touch 支持

- `touchstart` → 300ms timer
- `touchmove` → timer 到期后创建 floating，后续用 `elementFromPoint` 找目标，中线判定交换
- `touchend` → cleanup + `onSortEnd`

## Task 4: 集成到 app.js

- `initSortable()` → `enableSortable(todoList, { onSortEnd, isSameGroup })`
- `onSortEnd` → 按 newOrder 重建 todos → save + render
- `render()` 中每项调用 `bindSortableItem(li, sortableData)`

## Task 5: CSS

```css
.todo-item { transition: transform 0.25s ease; }
.drag-floating { transition: none !important; }
```

- 移动端长按 300ms 拖动
