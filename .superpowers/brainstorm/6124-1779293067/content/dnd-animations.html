Edge 交换 — 三种动画方式对比

桌面端用 HTML5 DnD，移动端用 Touch Events。选择动画策略

A

FLIP 动画

流程：记录 A 和 B 的 getBoundingClientRect() → 交换 DOM 位置 → 计算 delta → 设 transform 补偿 → 清除 transform 触发 transition → 两个元素同时对滑

优点：两个元素都有动画，视觉效果最流畅

风险：FLIP 计算须精确，如果同时还有其他 DOM 操作可能冲突

B

直接重渲染

流程：交换数组元素 → save() → render() → 整个列表重新生成 → CSS transition 让移动的元素平滑过渡

优点：代码最简单，永远和数组一致，不会出现 DOM 不同步

风险：全量渲染性能稍差，浮动克隆在 render 时会丢失，需要重新创建

C

混合：目标滑入 + 拖完统一 render

流程：拖拽中只动画被交换的目标元素（目标滑入源位置，源元素作为新 gap）→ 浮动克隆继续跟随指针 → touchend/drop 时统一 save + render

优点：动画简单（只移一个元素），不丢浮动克隆，最终渲染确保一致性

风险：交换次数多时，浮动位置和实际位置可能产生偏差
